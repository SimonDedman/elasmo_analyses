Implementation notes

  • Data source: the popover reads from the same docs/validate/assets/rules.json that already feeds the schema-level palette (after the 0ca2a672e commit it carries terms, threshold, anchors, case_sensitive_terms, _section_weights, and _proposal_url). No new data plumbing required.
  • Wiring: in validate.js, when rendering each column pill in _renderPaperRow(), append a <button class="rule-disc" data-col="...">?</button>. A single delegated click handler on the page reads data-col, looks up the rule in _rules[prefix][colName], and renders the popover.
  • Why per-column inline rather than only schema-level: the schema palette requires the validator to scroll past the rating UI to see rule details. The inline [?] answers "what does this column actually look for?" without leaving the rating row.
  • Estimated work: ~30 lines of JS, ~40 lines of CSS, no new data or build pipeline changes. Could land in a follow-up commit if the meeting endorses the pattern.
